1. CORE DESIGN SYSTEM & STYLING TOKENS (from index.css & Tailwind theme)
--------------------------------------------------------------------------------
A. Color Palette:
   - Primary: #004ac6 (Vibrant blue, used for accents, branding, and major action elements)
   - Primary Container: #2563eb (Royal blue, used for highlighted statuses and headers)
   - Primary Fixed: #dbe1ff (Light blue-grey highlight fill)
   - Primary Fixed Dim: #b4c5ff
   - Secondary: #515f74 (Slate grey-blue for secondary branding elements)
   - Secondary Container: #d5e3fc (Soft blue tint background for labels and badges)
   - Tertiary: #4d556b
   - Tertiary Container: #656d84
   - Surface / BG: #f7f9fb (Clean, soft light-grey background for overall pages)
   - Surface Bright: #f7f9fb
   - Surface Dim: #d8dadc
   - Surface Container: #eceef0
   - Surface Container Low: #f2f4f6
   - Surface Container High: #e6e8ea
   - Surface Container Highest: #e0e3e5
   - Surface Container Lowest: #ffffff (Pure white background for cards, forms, and overlays)
   - On Surface: #191c1e (Sleek dark grey/almost black for maximum readability)
   - On Surface Variant: #434655 (Muted grey for descriptions and labels)
   - Outline: #737686 (Border outline color)
   - Outline Variant: #c3c6d7 (Soft borders for cards and sections)
   - Dark Mode Accents (Login Gateway): #020617 (Rich slate dark) and #0f172a / #1e3a8a.

B. Typography:
   - Headline Font: "Hanken Grotesk", sans-serif (Bold, punchy, tracking-tight)
   - Body Font: "Inter", sans-serif (Highly readable, neutral, structured)
   - Icon Font: "Material Symbols Outlined" (Integrated via Google Fonts)
   - Size Scale: Base font scaled down to 92% (14.7px) on <html> for professional, dense layouts.
   - Text styling rules: Labels and action buttons frequently utilize capitalized uppercase text with spacing ('font-black text-[10px] tracking-widest' or 'tracking-[0.2em]').

C. Spacing & Borders:
   - Mobile Margin: 16px
   - Desktop Margin: 32px
   - Stack Spacing: Sm (8px), Md (16px), Lg (32px)
   - Border Radius: xl (0.5rem / 8px), lg (0.375rem / 6px)

D. Key Global Animations:
   - fadeIn: Standard page transition animation translating elements up by 10px smoothly.
   - shake: Error indication animation for form inputs.

--------------------------------------------------------------------------------
2. GENERAL COMPONENT & BUTTON STYLING PATTERNS
--------------------------------------------------------------------------------
- Primary Action Buttons (e.g., "Continue with Email", "Initialize Dashboard"):
  * Layout: Full-width or wide pill containers, h-14, black background (#111), rounded-2xl (16px).
  * Effects: Smooth transitions on active state (scale down to 98% and hover scaling up to 101%), with a spotlight hover sweep across the face (translateX from -100% to 100% using a primary-colored gradient light sweep).
  * Label: Uppercase, tracking-[0.2em], font-black, text-[10px], colored white.
- Secondary Buttons (e.g., "Continue with Google"):
  * Layout: Rounded-2xl (16px), h-14, pure white background, outline-variant border.
  * Hover: background surface change to lowest, shadow-[0_15px_35px_rgba(0,0,0,0.08)], slightly scaling Google logo.
- Steppers & Stepper Progress:
  * Uses 2px bg-surface-container track with bg-primary filling the progress bar (e.g., 50% or 75%).
  * Underlined with dense metadata: uppercase font-black step count (e.g., "Account", "Company", "Review").

--------------------------------------------------------------------------------
3. DETAILED SCREEN-BY-SCREEN ANALYSIS
--------------------------------------------------------------------------------

SCREEN 1: Identity Gateway (IdentityGateway.jsx)
- Purpose: Unified Auth Screen for both Recruiters and Job Seekers.
- Visuals: 
  * Split-screen background: Recruiter side uses very dark slates (#020617 / #0f172a), Seekers side uses vibrant blue (#004ac6 / #1e3a8a).
  * Floating Login Card: Center-aligned white glassmorphic container (bg-white/90 backdrop-blur-[40px]) with high-rim borders (border-white/40) and deep drop shadow.
  * Card Header: Heavy gradient (from primary via blue-700 to blue-900) containing the capitalized title "TALENT VECTOR".
- Interactive Elements:
  * Sliding Role Toggle: h-12 slider toggling active background between Recruiter and Job Seeker buttons.
  * Login Forms: Compact inputs for Email access, OTP 6-digit verification (with letters tracked at 0.5em spacing), and secure password setup.
  * Primary Button: "Continue with Email" / "Verify Code" / "Create Account & Continue".
  * Integration Button: "Continue with Google" (Oauth2 connector).

SCREEN 2: Recruiter Onboarding (RecruiterOnboarding.jsx)
- Purpose: Forced Company Profile Setup for first-time HR login.
- Visuals:
  * Clean, minimal header with blue branding.
  * Top progress bar at "Step 2 of 4" (50% Complete).
  * Central white container with "Company Profile" title and office building icon.
- Interactive Elements:
  * Required Input: Company Name (styled with a focus-ring inside an inner-shadowed input).
  * Optional Inputs: Website URL, Headquarters, Office Address.
  * Callout banner: Security notification in primary tint background (#004ac6 at 5% opacity, border-primary/10) locking profile to email.
  * Action Button: "Initialize Dashboard" (w-full, h-14, dark button).

SCREEN 3: Data Ingestion (DataIngestion.jsx)
- Purpose: Job Seeker CV upload page.
- Visuals:
  * Header with back-navigation arrow and circular profile avatar (which opens a popup logout menu on click).
  * Stepper indicating Step 2 of 4 (Upload CV).
  * Main Section split into form and document upload box.
- Interactive Elements:
  * Paste area: Large text-area box allowing raw CV copy-pasting.
  * Document drop area: Large dashed upload box. Changes styling (primary blue tint) when file is dragged or loaded.
  * Action Buttons: Sticky bottom footer showing "Skip" (secondary) and "Save and Process" (primary).

SCREEN 4: Profile Verification (ProfileVerification.jsx)
- Purpose: Displaying AI-parsed CV data for candidate validation (Human-in-the-Loop).
- Visuals:
  * 4-step progress stepper (Step 3 or 4 of 4).
  * Welcome Banner: bg-primary-container/10 border with verified check icon.
- State 1 (Verify Mode):
  * Left form layout: Editable inputs for Full Name, Email, Phone, Experience (Years), and Target Sector dropdown list.
  * Checklist card: Context checkrows with green verified badges.
  * Right column: Expertise Tags (skills displayed as removable badges with 'close' buttons). Search input with a dropdown popup to select from predefined skills database.
- State 2 (Review Mode):
  * Big green success celebration card (bg-emerald-50, border-emerald-200) saying "Profile Ready!".
  * Two review columns: Identity metrics (large typography) and Top Skills tags.
- Sticky Footer: "Back to Edit" / "Re-upload" (left) and "Confirm Details" / "Enter Dashboard" (right).

SCREEN 5: JDSourcing (JDSourcing.jsx)
- Purpose: Job Description input, metadata editing (HITL), and dual-sourcing triggers.
- Visuals:
  * Progress steps: (1) Upload JD, (2) Review Metadata, (3) Choose Track.
- State 1 (Upload):
  * Huge dashed upload box with cloud_upload icon. Processes files on selection.
- State 2 (Review / HITL):
  * Grid form containing Job Title, Min Experience, Location Type (Onsite/Remote/Hybrid select dropdown), and Company Email.
  * Core Skills Area: Extracted skills rendered as badges with close buttons, plus an interactive input to type and add new tags dynamically.
  * Button: "Finalize and Continue".
- State 3 (Select Strategy & Dual Match Results):
  * Split layouts displaying two main strategy cards:
    1. "Private Bulk Upload": Triggers a hidden file selector for local resume files. Saves candidate files under visibility: "private".
    2. "Global Talent Pool": Triggers a backend query scanning all registered public profiles.
  * Top Candidates list: Cards with Rank Number, candidate name, exp, matching percentage, tags, and a circular "Invite to Interview" button with send icon.

SCREEN 6: Recruiter Portal (RecruiterPortal.jsx)
- Purpose: Unified workspace to run fast, ad-hoc resume screening.
- Visuals:
  * Hero Section: "Recruiter Intelligence" title on the left, "New JD Analysis" icon button on the right.
  * Left Column (Configuration):
    * Job Description input: Large text area + file uploader box.
    * Resume Batch box: Dashed file selector area displaying file count.
    * Action button: "Start AI Screening" (with a spinning sync icon during loader states).
  * Right Column (Results):
    * Shows a placeholder graphic ("No Analysis Results" with analytics icon) if empty.
    * Renders "Top Candidates" list sorted by rank, featuring experience details, skill tags, match percentage, and a download icon.

SCREEN 7: Market Discovery (MarketDiscovery.jsx)
- Purpose: Job Seeker job board/listings.
- Visuals:
  * Top header with profile picture, notifications, and brand name.
  * Filter bar: Horizontal-scrolling pill buttons representing sectors (All Jobs, Engineering, Design, etc.). Active filter turns blue.
  * Job Cards: Large list items with company logo, title (which offsets to the right on hover), salary and shift tags, and bookmark icon.
- Navigation:
  * Floating bottom navigation bar: Rounded glass pill (bg-white/80, backdrop-blur-3xl) with 4 links: Home, Jobs (active with an underline indicator), Profile, Settings.

SCREEN 8: User Profile (UserProfile.jsx)
- Purpose: Job Seeker profile editing.
- Visuals:
  * Top header: Back-arrow button, screen title, and "Save Changes" save icon button.
  * Left Column: Profile Card showing user initials, name, email, sector, and experience. Also contains an update CV file upload button.
  * Right Column: Core information edit inputs (Name, Email read-only, Phone, Experience, Sector) and Expertise skills tag list with search dropdown autocomplete.
  * Overlays: A premium central checkmark modal saying "Changes Saved" appearing on successful save.
